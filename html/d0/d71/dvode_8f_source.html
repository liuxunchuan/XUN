17 C-----------------------------------------------------------------------
18 C DVODE.. Variable-coefficient Ordinary Differential Equation solver,
19 C with fixed-leading-coefficient implementation.
20 C This version is in double precision.
21 C
22 C DVODE solves the initial value problem for stiff or nonstiff
23 C systems of first order ODEs,
24 C dy/dt = f(t,y) , or, in component form,
25 C dy(i)/dt = f(i) = f(i,t,y(1),y(2),...,y(NEQ)) (i = 1,...,NEQ).
26 C DVODE is a package based on the EPISODE and EPISODEB packages, and
27 C on the ODEPACK user interface standard, with minor modifications.

80 C Summary of usage.
81 C
82 C Communication between the user and the DVODE package, for normal
83 C situations, is summarized here. This summary describes only a subset
84 C of the full set of options available. See the full description for
85 C details, including optional communication, nonstandard options,
86 C and instructions for special situations. See also the example
87 C problem (with program and output) following this summary.
88 C
89 C A. First provide a subroutine of the form..
90 C
91 C SUBROUTINE F (NEQ, T, Y, YDOT, RPAR, IPAR)
92 C DOUBLE PRECISION T, Y, YDOT, RPAR
93 C DIMENSION Y(NEQ), YDOT(NEQ)
94 C
95 C which supplies the vector function f by loading YDOT(i) with f(i).
96 C
97 C B. Next determine (or guess) whether or not the problem is stiff.
98 C Stiffness occurs when the Jacobian matrix df/dy has an eigenvalue
99 C whose real part is negative and large in magnitude, compared to the
100 C reciprocal of the t span of interest. If the problem is nonstiff,
101 C use a method flag MF = 10. If it is stiff, there are four standard
102 C choices for MF (21, 22, 24, 25), and DVODE requires the Jacobian
103 C matrix in some form. In these cases (MF .gt. 0), DVODE will use a
104 C saved copy of the Jacobian matrix. If this is undesirable because of
105 C storage limitations, set MF to the corresponding negative value
106 C (-21, -22, -24, -25). (See full description of MF below.)
107 C The Jacobian matrix is regarded either as full (MF = 21 or 22),
108 C or banded (MF = 24 or 25). In the banded case, DVODE requires two
109 C half-bandwidth parameters ML and MU. These are, respectively, the
110 C widths of the lower and upper parts of the band, excluding the main
111 C diagonal. Thus the band consists of the locations (i,j) with
112 C i-ML .le. j .le. i+MU, and the full bandwidth is ML+MU+1.
113 C
114 C C. If the problem is stiff, you are encouraged to supply the Jacobian
115 C directly (MF = 21 or 24), but if this is not feasible, DVODE will
116 C compute it internally by difference quotients (MF = 22 or 25).
117 C If you are supplying the Jacobian, provide a subroutine of the form..
118 C
119 C SUBROUTINE JAC (NEQ, T, Y, ML, MU, PD, NROWPD, RPAR, IPAR)
120 C DOUBLE PRECISION T, Y, PD, RPAR
121 C DIMENSION Y(NEQ), PD(NROWPD,NEQ)
122 C
123 C which supplies df/dy by loading PD as follows..
124 C For a full Jacobian (MF = 21), load PD(i,j) with df(i)/dy(j),
125 C the partial derivative of f(i) with respect to y(j). (Ignore the
126 C ML and MU arguments in this case.)
127 C For a banded Jacobian (MF = 24), load PD(i-j+MU+1,j) with
128 C df(i)/dy(j), i.e. load the diagonal lines of df/dy into the rows of
129 C PD from the top down.
130 C In either case, only nonzero elements need be loaded.
131 C
132 C D. Write a main program which calls subroutine DVODE once for
133 C each point at which answers are desired. This should also provide
134 C for possible use of logical unit 6 for output of error messages
135 C by DVODE. On the first call to DVODE, supply arguments as follows..
136 C F = Name of subroutine for right-hand side vector f.
137 C This name must be declared external in calling program.
138 C NEQ = Number of first order ODE-s.
139 C Y = Array of initial values, of length NEQ.
140 C T = The initial value of the independent variable.
141 C TOUT = First point where output is desired (.ne. T).
142 C ITOL = 1 or 2 according as ATOL (below) is a scalar or array.
143 C RTOL = Relative tolerance parameter (scalar).
144 C ATOL = Absolute tolerance parameter (scalar or array).
145 C The estimated local error in Y(i) will be controlled so as
146 C to be roughly less (in magnitude) than
147 C EWT(i) = RTOL*abs(Y(i)) + ATOL if ITOL = 1, or
148 C EWT(i) = RTOL*abs(Y(i)) + ATOL(i) if ITOL = 2.
149 C Thus the local error test passes if, in each component,
150 C either the absolute error is less than ATOL (or ATOL(i)),
151 C or the relative error is less than RTOL.
152 C Use RTOL = 0.0 for pure absolute error control, and
153 C use ATOL = 0.0 (or ATOL(i) = 0.0) for pure relative error
154 C control. Caution.. Actual (global) errors may exceed these
155 C local tolerances, so choose them conservatively.
156 C ITASK = 1 for normal computation of output values of Y at t = TOUT.
157 C ISTATE = Integer flag (input and output). Set ISTATE = 1.
158 C IOPT = 0 to indicate no optional input used.
159 C RWORK = Real work array of length at least..
160 C 20 + 16*NEQ for MF = 10,
161 C 22 + 9*NEQ + 2*NEQ**2 for MF = 21 or 22,
162 C 22 + 11*NEQ + (3*ML + 2*MU)*NEQ for MF = 24 or 25.
163 C LRW = Declared length of RWORK (in user's DIMENSION statement).
164 C IWORK = Integer work array of length at least..
165 C 30 for MF = 10,
166 C 30 + NEQ for MF = 21, 22, 24, or 25.
167 C If MF = 24 or 25, input in IWORK(1),IWORK(2) the lower
168 C and upper half-bandwidths ML,MU.
169 C LIW = Declared length of IWORK (in user's DIMENSION statement).
170 C JAC = Name of subroutine for Jacobian matrix (MF = 21 or 24).
171 C If used, this name must be declared external in calling
172 C program. If not used, pass a dummy name.
173 C MF = Method flag. Standard values are..
174 C 10 for nonstiff (Adams) method, no Jacobian used.
175 C 21 for stiff (BDF) method, user-supplied full Jacobian.
176 C 22 for stiff method, internally generated full Jacobian.
177 C 24 for stiff method, user-supplied banded Jacobian.
178 C 25 for stiff method, internally generated banded Jacobian.
179 C RPAR,IPAR = user-defined real and integer arrays passed to F and JAC.
180 C Note that the main program must declare arrays Y, RWORK, IWORK,
181 C and possibly ATOL, RPAR, and IPAR.
182 C
183 C E. The output from the first call (or any call) is..
184 C Y = Array of computed values of y(t) vector.
185 C T = Corresponding value of independent variable (normally TOUT).
186 C ISTATE = 2 if DVODE was successful, negative otherwise.
187 C -1 means excess work done on this call. (Perhaps wrong MF.)
188 C -2 means excess accuracy requested. (Tolerances too small.)
189 C -3 means illegal input detected. (See printed message.)
190 C -4 means repeated error test failures. (Check all input.)
191 C -5 means repeated convergence failures. (Perhaps bad
192 C Jacobian supplied or wrong choice of MF or tolerances.)
193 C -6 means error weight became zero during problem. (Solution
194 C component i vanished, and ATOL or ATOL(i) = 0.)
195 C
196 C F. To continue the integration after a successful return, simply
197 C reset TOUT and call DVODE again. No other parameters need be reset.
198 C

303 C Full description of user interface to DVODE.
304 C
305 C The user interface to DVODE consists of the following parts.
306 C
307 C i. The call sequence to subroutine DVODE, which is a driver
308 C routine for the solver. This includes descriptions of both
309 C the call sequence arguments and of user-supplied routines.
310 C Following these descriptions is
311 C * a description of optional input available through the
312 C call sequence,
313 C * a description of optional output (in the work arrays), and
314 C * instructions for interrupting and restarting a solution.
315 C
316 C ii. Descriptions of other routines in the DVODE package that may be
317 C (optionally) called by the user. These provide the ability to
318 C alter error message handling, save and restore the internal
319 C COMMON, and obtain specified derivatives of the solution y(t).
320 C
321 C iii. Descriptions of COMMON blocks to be declared in overlay
322 C or similar environments.
323 C
324 C iv. Description of two routines in the DVODE package, either of
325 C which the user may replace with his own version, if desired.
326 C these relate to the measurement of errors.
327 C
328 C-----------------------------------------------------------------------
329 C Part i. Call Sequence.
330 C
331 C The call sequence parameters used for input only are
332 C F, NEQ, TOUT, ITOL, RTOL, ATOL, ITASK, IOPT, LRW, LIW, JAC, MF,
333 C and those used for both input and output are
334 C Y, T, ISTATE.
335 C The work arrays RWORK and IWORK are also used for conditional and
336 C optional input and optional output. (The term output here refers
337 C to the return from subroutine DVODE to the user's calling program.)
338 C
339 C The legality of input parameters will be thoroughly checked on the
340 C initial call for the problem, but not checked thereafter unless a
341 C change in input parameters is flagged by ISTATE = 3 in the input.
342 C
343 C The descriptions of the call arguments are as follows.
344 C
345 C F = The name of the user-supplied subroutine defining the
346 C ODE system. The system must be put in the first-order
347 C form dy/dt = f(t,y), where f is a vector-valued function
348 C of the scalar t and the vector y. Subroutine F is to
349 C compute the function f. It is to have the form
350 C SUBROUTINE F (NEQ, T, Y, YDOT, RPAR, IPAR)
351 C DOUBLE PRECISION T, Y, YDOT, RPAR
352 C DIMENSION Y(NEQ), YDOT(NEQ)
353 C where NEQ, T, and Y are input, and the array YDOT = f(t,y)
354 C is output. Y and YDOT are arrays of length NEQ.
355 C (In the DIMENSION statement above, NEQ can be replaced by
356 C * to make Y and YDOT assumed size arrays.)
357 C Subroutine F should not alter Y(1),...,Y(NEQ).
358 C F must be declared EXTERNAL in the calling program.
359 C
360 C Subroutine F may access user-defined real and integer
361 C work arrays RPAR and IPAR, which are to be dimensioned
362 C in the main program.
363 C
364 C If quantities computed in the F routine are needed
365 C externally to DVODE, an extra call to F should be made
366 C for this purpose, for consistent and accurate results.
367 C If only the derivative dy/dt is needed, use DVINDY instead.
368 C
369 C NEQ = The size of the ODE system (number of first order
370 C ordinary differential equations). Used only for input.
371 C NEQ may not be increased during the problem, but
372 C can be decreased (with ISTATE = 3 in the input).
373 C
374 C Y = A real array for the vector of dependent variables, of
375 C length NEQ or more. Used for both input and output on the
376 C first call (ISTATE = 1), and only for output on other calls.
377 C On the first call, Y must contain the vector of initial
378 C values. In the output, Y contains the computed solution
379 C evaluated at T. If desired, the Y array may be used
380 C for other purposes between calls to the solver.
381 C
382 C This array is passed as the Y argument in all calls to
383 C F and JAC.
384 C
385 C T = The independent variable. In the input, T is used only on
386 C the first call, as the initial point of the integration.
387 C In the output, after each call, T is the value at which a
388 C computed solution Y is evaluated (usually the same as TOUT).
389 C On an error return, T is the farthest point reached.
390 C
391 C TOUT = The next value of t at which a computed solution is desired.
392 C Used only for input.
393 C
394 C When starting the problem (ISTATE = 1), TOUT may be equal
395 C to T for one call, then should .ne. T for the next call.
396 C For the initial T, an input value of TOUT .ne. T is used
397 C in order to determine the direction of the integration
398 C (i.e. the algebraic sign of the step sizes) and the rough
399 C scale of the problem. Integration in either direction
400 C (forward or backward in t) is permitted.
401 C
402 C If ITASK = 2 or 5 (one-step modes), TOUT is ignored after
403 C the first call (i.e. the first call with TOUT .ne. T).
404 C Otherwise, TOUT is required on every call.
405 C
406 C If ITASK = 1, 3, or 4, the values of TOUT need not be
407 C monotone, but a value of TOUT which backs up is limited
408 C to the current internal t interval, whose endpoints are
409 C TCUR - HU and TCUR. (See optional output, below, for
410 C TCUR and HU.)
411 C
412 C ITOL = An indicator for the type of error control. See
413 C description below under ATOL. Used only for input.
414 C
415 C RTOL = A relative error tolerance parameter, either a scalar or
416 C an array of length NEQ. See description below under ATOL.
417 C Input only.
418 C
419 C ATOL = An absolute error tolerance parameter, either a scalar or
420 C an array of length NEQ. Input only.
421 C
422 C The input parameters ITOL, RTOL, and ATOL determine
423 C the error control performed by the solver. The solver will
424 C control the vector e = (e(i)) of estimated local errors
425 C in Y, according to an inequality of the form
426 C rms-norm of ( e(i)/EWT(i) ) .le. 1,
427 C where EWT(i) = RTOL(i)*abs(Y(i)) + ATOL(i),
428 C and the rms-norm (root-mean-square norm) here is
429 C rms-norm(v) = sqrt(sum v(i)**2 / NEQ). Here EWT = (EWT(i))
430 C is a vector of weights which must always be positive, and
431 C the values of RTOL and ATOL should all be non-negative.
432 C The following table gives the types (scalar/array) of
433 C RTOL and ATOL, and the corresponding form of EWT(i).
434 C
435 C ITOL RTOL ATOL EWT(i)
436 C 1 scalar scalar RTOL*ABS(Y(i)) + ATOL
437 C 2 scalar array RTOL*ABS(Y(i)) + ATOL(i)
438 C 3 array scalar RTOL(i)*ABS(Y(i)) + ATOL
439 C 4 array array RTOL(i)*ABS(Y(i)) + ATOL(i)
440 C
441 C When either of these parameters is a scalar, it need not
442 C be dimensioned in the user's calling program.
443 C
444 C If none of the above choices (with ITOL, RTOL, and ATOL
445 C fixed throughout the problem) is suitable, more general
446 C error controls can be obtained by substituting
447 C user-supplied routines for the setting of EWT and/or for
448 C the norm calculation. See Part iv below.
449 C
450 C If global errors are to be estimated by making a repeated
451 C run on the same problem with smaller tolerances, then all
452 C components of RTOL and ATOL (i.e. of EWT) should be scaled
453 C down uniformly.
454 C
455 C ITASK = An index specifying the task to be performed.
456 C Input only. ITASK has the following values and meanings.
457 C 1 means normal computation of output values of y(t) at
458 C t = TOUT (by overshooting and interpolating).
459 C 2 means take one step only and return.
460 C 3 means stop at the first internal mesh point at or
461 C beyond t = TOUT and return.
462 C 4 means normal computation of output values of y(t) at
463 C t = TOUT but without overshooting t = TCRIT.
464 C TCRIT must be input as RWORK(1). TCRIT may be equal to
465 C or beyond TOUT, but not behind it in the direction of
466 C integration. This option is useful if the problem
467 C has a singularity at or beyond t = TCRIT.
468 C 5 means take one step, without passing TCRIT, and return.
469 C TCRIT must be input as RWORK(1).
470 C
471 C Note.. If ITASK = 4 or 5 and the solver reaches TCRIT
472 C (within roundoff), it will return T = TCRIT (exactly) to
473 C indicate this (unless ITASK = 4 and TOUT comes before TCRIT,
474 C in which case answers at T = TOUT are returned first).
475 C
476 C ISTATE = an index used for input and output to specify the
477 C the state of the calculation.
478 C
479 C In the input, the values of ISTATE are as follows.
480 C 1 means this is the first call for the problem
481 C (initializations will be done). See note below.
482 C 2 means this is not the first call, and the calculation
483 C is to continue normally, with no change in any input
484 C parameters except possibly TOUT and ITASK.
485 C (If ITOL, RTOL, and/or ATOL are changed between calls
486 C with ISTATE = 2, the new values will be used but not
487 C tested for legality.)
488 C 3 means this is not the first call, and the
489 C calculation is to continue normally, but with
490 C a change in input parameters other than
491 C TOUT and ITASK. Changes are allowed in
492 C NEQ, ITOL, RTOL, ATOL, IOPT, LRW, LIW, MF, ML, MU,
493 C and any of the optional input except H0.
494 C (See IWORK description for ML and MU.)
495 C Note.. A preliminary call with TOUT = T is not counted
496 C as a first call here, as no initialization or checking of
497 C input is done. (Such a call is sometimes useful to include
498 C the initial conditions in the output.)
499 C Thus the first call for which TOUT .ne. T requires
500 C ISTATE = 1 in the input.
501 C
502 C In the output, ISTATE has the following values and meanings.
503 C 1 means nothing was done, as TOUT was equal to T with
504 C ISTATE = 1 in the input.
505 C 2 means the integration was performed successfully.
506 C -1 means an excessive amount of work (more than MXSTEP
507 C steps) was done on this call, before completing the
508 C requested task, but the integration was otherwise
509 C successful as far as T. (MXSTEP is an optional input
510 C and is normally 500.) To continue, the user may
511 C simply reset ISTATE to a value .gt. 1 and call again.
512 C (The excess work step counter will be reset to 0.)
513 C In addition, the user may increase MXSTEP to avoid
514 C this error return. (See optional input below.)
515 C -2 means too much accuracy was requested for the precision
516 C of the machine being used. This was detected before
517 C completing the requested task, but the integration
518 C was successful as far as T. To continue, the tolerance
519 C parameters must be reset, and ISTATE must be set
520 C to 3. The optional output TOLSF may be used for this
521 C purpose. (Note.. If this condition is detected before
522 C taking any steps, then an illegal input return
523 C (ISTATE = -3) occurs instead.)
524 C -3 means illegal input was detected, before taking any
525 C integration steps. See written message for details.
526 C Note.. If the solver detects an infinite loop of calls
527 C to the solver with illegal input, it will cause
528 C the run to stop.
529 C -4 means there were repeated error test failures on
530 C one attempted step, before completing the requested
531 C task, but the integration was successful as far as T.
532 C The problem may have a singularity, or the input
533 C may be inappropriate.
534 C -5 means there were repeated convergence test failures on
535 C one attempted step, before completing the requested
536 C task, but the integration was successful as far as T.
537 C This may be caused by an inaccurate Jacobian matrix,
538 C if one is being used.
539 C -6 means EWT(i) became zero for some i during the
540 C integration. Pure relative error control (ATOL(i)=0.0)
541 C was requested on a variable which has now vanished.
542 C The integration was successful as far as T.
543 C
544 C Note.. Since the normal output value of ISTATE is 2,
545 C it does not need to be reset for normal continuation.
546 C Also, since a negative input value of ISTATE will be
547 C regarded as illegal, a negative output value requires the
548 C user to change it, and possibly other input, before
549 C calling the solver again.
550 C
551 C IOPT = An integer flag to specify whether or not any optional
552 C input is being used on this call. Input only.
553 C The optional input is listed separately below.
554 C IOPT = 0 means no optional input is being used.
555 C Default values will be used in all cases.
556 C IOPT = 1 means optional input is being used.
557 C
558 C RWORK = A real working array (double precision).
559 C The length of RWORK must be at least
560 C 20 + NYH*(MAXORD + 1) + 3*NEQ + LWM where
561 C NYH = the initial value of NEQ,
562 C MAXORD = 12 (if METH = 1) or 5 (if METH = 2) (unless a
563 C smaller value is given as an optional input),
564 C LWM = length of work space for matrix-related data..
565 C LWM = 0 if MITER = 0,
566 C LWM = 2*NEQ**2 + 2 if MITER = 1 or 2, and MF.gt.0,
567 C LWM = NEQ**2 + 2 if MITER = 1 or 2, and MF.lt.0,
568 C LWM = NEQ + 2 if MITER = 3,
569 C LWM = (3*ML+2*MU+2)*NEQ + 2 if MITER = 4 or 5, and MF.gt.0,
570 C LWM = (2*ML+MU+1)*NEQ + 2 if MITER = 4 or 5, and MF.lt.0.
571 C (See the MF description for METH and MITER.)
572 C Thus if MAXORD has its default value and NEQ is constant,
573 C this length is..
574 C 20 + 16*NEQ for MF = 10,
575 C 22 + 16*NEQ + 2*NEQ**2 for MF = 11 or 12,
576 C 22 + 16*NEQ + NEQ**2 for MF = -11 or -12,
577 C 22 + 17*NEQ for MF = 13,
578 C 22 + 18*NEQ + (3*ML+2*MU)*NEQ for MF = 14 or 15,
579 C 22 + 17*NEQ + (2*ML+MU)*NEQ for MF = -14 or -15,
580 C 20 + 9*NEQ for MF = 20,
581 C 22 + 9*NEQ + 2*NEQ**2 for MF = 21 or 22,
582 C 22 + 9*NEQ + NEQ**2 for MF = -21 or -22,
583 C 22 + 10*NEQ for MF = 23,
584 C 22 + 11*NEQ + (3*ML+2*MU)*NEQ for MF = 24 or 25.
585 C 22 + 10*NEQ + (2*ML+MU)*NEQ for MF = -24 or -25.
586 C The first 20 words of RWORK are reserved for conditional
587 C and optional input and optional output.
588 C
589 C The following word in RWORK is a conditional input..
590 C RWORK(1) = TCRIT = critical value of t which the solver
591 C is not to overshoot. Required if ITASK is
592 C 4 or 5, and ignored otherwise. (See ITASK.)
593 C
594 C LRW = The length of the array RWORK, as declared by the user.
595 C (This will be checked by the solver.)
596 C
597 C IWORK = An integer work array. The length of IWORK must be at least
598 C 30 if MITER = 0 or 3 (MF = 10, 13, 20, 23), or
599 C 30 + NEQ otherwise (abs(MF) = 11,12,14,15,21,22,24,25).
600 C The first 30 words of IWORK are reserved for conditional and
601 C optional input and optional output.
602 C
603 C The following 2 words in IWORK are conditional input..
604 C IWORK(1) = ML These are the lower and upper
605 C IWORK(2) = MU half-bandwidths, respectively, of the
606 C banded Jacobian, excluding the main diagonal.
607 C The band is defined by the matrix locations
608 C (i,j) with i-ML .le. j .le. i+MU. ML and MU
609 C must satisfy 0 .le. ML,MU .le. NEQ-1.
610 C These are required if MITER is 4 or 5, and
611 C ignored otherwise. ML and MU may in fact be
612 C the band parameters for a matrix to which
613 C df/dy is only approximately equal.
614 C
615 C LIW = the length of the array IWORK, as declared by the user.
616 C (This will be checked by the solver.)
617 C
618 C Note.. The work arrays must not be altered between calls to DVODE
619 C for the same problem, except possibly for the conditional and
620 C optional input, and except for the last 3*NEQ words of RWORK.
621 C The latter space is used for internal scratch space, and so is
622 C available for use by the user outside DVODE between calls, if
623 C desired (but not for use by F or JAC).
624 C
625 C JAC = The name of the user-supplied routine (MITER = 1 or 4) to
626 C compute the Jacobian matrix, df/dy, as a function of
627 C the scalar t and the vector y. It is to have the form
628 C SUBROUTINE JAC (NEQ, T, Y, ML, MU, PD, NROWPD,
629 C RPAR, IPAR)
630 C DOUBLE PRECISION T, Y, PD, RPAR
631 C DIMENSION Y(NEQ), PD(NROWPD, NEQ)
632 C where NEQ, T, Y, ML, MU, and NROWPD are input and the array
633 C PD is to be loaded with partial derivatives (elements of the
634 C Jacobian matrix) in the output. PD must be given a first
635 C dimension of NROWPD. T and Y have the same meaning as in
636 C Subroutine F. (In the DIMENSION statement above, NEQ can
637 C be replaced by * to make Y and PD assumed size arrays.)
638 C In the full matrix case (MITER = 1), ML and MU are
639 C ignored, and the Jacobian is to be loaded into PD in
640 C columnwise manner, with df(i)/dy(j) loaded into PD(i,j).
641 C In the band matrix case (MITER = 4), the elements
642 C within the band are to be loaded into PD in columnwise
643 C manner, with diagonal lines of df/dy loaded into the rows
644 C of PD. Thus df(i)/dy(j) is to be loaded into PD(i-j+MU+1,j).
645 C ML and MU are the half-bandwidth parameters. (See IWORK).
646 C The locations in PD in the two triangular areas which
647 C correspond to nonexistent matrix elements can be ignored
648 C or loaded arbitrarily, as they are overwritten by DVODE.
649 C JAC need not provide df/dy exactly. A crude
650 C approximation (possibly with a smaller bandwidth) will do.
651 C In either case, PD is preset to zero by the solver,
652 C so that only the nonzero elements need be loaded by JAC.
653 C Each call to JAC is preceded by a call to F with the same
654 C arguments NEQ, T, and Y. Thus to gain some efficiency,
655 C intermediate quantities shared by both calculations may be
656 C saved in a user COMMON block by F and not recomputed by JAC,
657 C if desired. Also, JAC may alter the Y array, if desired.
658 C JAC must be declared external in the calling program.
659 C Subroutine JAC may access user-defined real and integer
660 C work arrays, RPAR and IPAR, whose dimensions are set by the
661 C user in the main program.
662 C
663 C MF = The method flag. Used only for input. The legal values of
664 C MF are 10, 11, 12, 13, 14, 15, 20, 21, 22, 23, 24, 25,
665 C -11, -12, -14, -15, -21, -22, -24, -25.
666 C MF is a signed two-digit integer, MF = JSV*(10*METH + MITER).
667 C JSV = SIGN(MF) indicates the Jacobian-saving strategy..
668 C JSV = 1 means a copy of the Jacobian is saved for reuse
669 C in the corrector iteration algorithm.
670 C JSV = -1 means a copy of the Jacobian is not saved
671 C (valid only for MITER = 1, 2, 4, or 5).
672 C METH indicates the basic linear multistep method..
673 C METH = 1 means the implicit Adams method.
674 C METH = 2 means the method based on backward
675 C differentiation formulas (BDF-s).
676 C MITER indicates the corrector iteration method..
677 C MITER = 0 means functional iteration (no Jacobian matrix
678 C is involved).
679 C MITER = 1 means chord iteration with a user-supplied
680 C full (NEQ by NEQ) Jacobian.
681 C MITER = 2 means chord iteration with an internally
682 C generated (difference quotient) full Jacobian
683 C (using NEQ extra calls to F per df/dy value).
684 C MITER = 3 means chord iteration with an internally
685 C generated diagonal Jacobian approximation
686 C (using 1 extra call to F per df/dy evaluation).
687 C MITER = 4 means chord iteration with a user-supplied
688 C banded Jacobian.
689 C MITER = 5 means chord iteration with an internally
690 C generated banded Jacobian (using ML+MU+1 extra
691 C calls to F per df/dy evaluation).
692 C If MITER = 1 or 4, the user must supply a subroutine JAC
693 C (the name is arbitrary) as described above under JAC.
694 C For other values of MITER, a dummy argument can be used.
695 C
696 C RPAR User-specified array used to communicate real parameters
697 C to user-supplied subroutines. If RPAR is a vector, then
698 C it must be dimensioned in the user's main program. If it
699 C is unused or it is a scalar, then it need not be
700 C dimensioned.
701 C
702 C IPAR User-specified array used to communicate integer parameter
703 C to user-supplied subroutines. The comments on dimensioning
704 C RPAR apply to IPAR.
705 C-----------------------------------------------------------------------
706 C Optional Input.
707 C
708 C The following is a list of the optional input provided for in the

710 C this table lists its name as used in this documentation, its

898 C The detailed instructions for using DVINDY are as follows.

900 C

902 C

916 C NYH = Column length of YH, equal to the initial value of NEQ.
917 C
918 C The output parameters are..
919 C
920 C DKY = A real array of length NEQ containing the computed value
921 C of the K-th derivative of y(t).
922 C IFLAG = Integer flag, returned as 0 if K and T were legal,

941 C-----------------------------------------------------------------------
942 C Part iv. Optionally Replaceable Solver Routines.
943 C
944 C Below are descriptions of two routines in the DVODE package which
945 C relate to the measurement of errors. Either routine can be
946 C replaced by a user-supplied version, if desired. However, since such
947 C a replacement may have a major impact on performance, it should be
948 C done only when absolutely necessary, and only with great caution.
949 C (Note.. The means by which the package version of a routine is
950 C superseded by the user's version may be system-dependent.)
951 C
952 C (a) DEWSET.
953 C The following subroutine is called just before each internal
954 C integration step, and sets the array of error weights, EWT, as
955 C described under ITOL/RTOL/ATOL above..
956 C SUBROUTINE DEWSET (NEQ, ITOL, RTOL, ATOL, YCUR, EWT)
957 C where NEQ, ITOL, RTOL, and ATOL are as in the DVODE call sequence,
958 C YCUR contains the current dependent variable vector, and
959 C EWT is the array of weights set by DEWSET.
960 C
961 C If the user supplies this subroutine, it must return in EWT(i)
962 C (i = 1,...,NEQ) a positive quantity suitable for comparison with
963 C errors in Y(i). The EWT array returned by DEWSET is passed to the
964 C DVNORM routine (See below.), and also used by DVODE in the computation
965 C of the optional output IMXER, the diagonal Jacobian approximation,
966 C and the increments for difference quotient Jacobians.
967 C
968 C In the user-supplied version of DEWSET, it may be desirable to use
969 C the current values of derivatives of y. Derivatives up to order NQ
970 C are available from the history array YH, described above under
971 C Optional Output. In DEWSET, YH is identical to the YCUR array,
972 C extended to NQ + 1 columns with a column length of NYH and scale
973 C factors of h**j/factorial(j). On the first call for the problem,
974 C given by NST = 0, NQ is 1 and H is temporarily set to 1.0.
975 C NYH is the initial value of NEQ. The quantities NQ, H, and NST
976 C can be obtained by including in DEWSET the statements..

1689  SUBROUTINE dvhin (N, T0, Y0, YDOT, F, RPAR, IPAR, TOUT, UROUND,
1690  1 EWT, ITOL, ATOL, Y, TEMP, H0, NITER, IER)
1691  EXTERNAL f
1692  DOUBLE PRECISION t0, y0, ydot, rpar, tout, uround, ewt, atol, y,
1693  1 temp, h0
1694  INTEGER n, ipar, itol, niter, ier
1695  dimension y0(*), ydot(*), ewt(*), atol(*), y(*),
1696  1 temp(*), rpar(*), ipar(*)
1697 C-----------------------------------------------------------------------
1698 C Call sequence input -- N, T0, Y0, YDOT, F, RPAR, IPAR, TOUT, UROUND,
1699 C EWT, ITOL, ATOL, Y, TEMP
1700 C Call sequence output -- H0, NITER, IER
1701 C COMMON block variables accessed -- None
1702 C
1703 C Subroutines called by DVHIN.. F
1704 C Function routines called by DVHIN.. DVNORM
1705 C-----------------------------------------------------------------------
1706 C This routine computes the step size, H0, to be attempted on the
1707 C first step, when the user has not supplied a value for this.
1708 C
1709 C First we check that TOUT - T0 differs significantly from zero. Then
1710 C an iteration is done to approximate the initial second derivative
1711 C and this is used to define h from w.r.m.s.norm(h**2 * yddot / 2) = 1.
1712 C A bias factor of 1/2 is applied to the resulting h.
1713 C The sign of H0 is inferred from the initial values of TOUT and T0.
1714 C
1715 C Communication with DVHIN is done with the following variables..
1716 C
1717 C N = Size of ODE system, input.
1718 C T0 = Initial value of independent variable, input.
1719 C Y0 = Vector of initial conditions, input.
1720 C YDOT = Vector of initial first derivatives, input.
1721 C F = Name of subroutine for right-hand side f(t,y), input.
1722 C RPAR, IPAR = Dummy names for user's real and integer work arrays.
1723 C TOUT = First output value of independent variable
1724 C UROUND = Machine unit roundoff
1725 C EWT, ITOL, ATOL = Error weights and tolerance parameters
1726 C as described in the driver routine, input.
1727 C Y, TEMP = Work arrays of length N.
1728 C H0 = Step size to be attempted, output.
1729 C NITER = Number of iterations (and of f evaluations) to compute H0,
1730 C output.
1731 C IER = The error flag, returned with the value
1732 C IER = 0 if no trouble occurred, or
1733 C IER = -1 if TOUT and T0 are considered too close to proceed.
1734 C-----------------------------------------------------------------------
1735 C
1736 C Type declarations for local variables --------------------------------
1737 C
1738  DOUBLE PRECISION afi, atoli, delyi, h, half, hg, hlb, hnew, hrat,
1739  1 hub, hun, pt1, t1, tdist, tround, two, yddnrm
1740  INTEGER i, iter
1741 C

1743 C
1744  DOUBLE PRECISION dvnorm
1745 C-----------------------------------------------------------------------
1746 C The following Fortran-77 declaration is to cause the values of the
1747 C listed (local) variables to be saved between calls to this integrator.
1748 C-----------------------------------------------------------------------
1749  SAVE half, hun, pt1, two
1750  DATA half /0.5d0/, hun /100.0d0/, pt1 /0.1d0/, two /2.0d0/
1751 C
1752  niter = 0
1753  tdist = abs(tout - t0)
1754  tround = uround*max(abs(t0),abs(tout))
1755  IF (tdist .LT. two*tround) GO TO 100
1756 C
1757 C Set a lower bound on h based on the roundoff level in T0 and TOUT. ---
1758  hlb = hun*tround
1759 C Set an upper bound on h based on TOUT-T0 and the initial Y and YDOT. -
1760  hub = pt1*tdist
1761  atoli = atol(1)
1762  DO 10 i = 1, n
1763  IF (itol .EQ. 2 .OR. itol .EQ. 4) atoli = atol(i)
1764  delyi = pt1*abs(y0(i)) + atoli
1765  afi = abs(ydot(i))
1766  IF (afi*hub .GT. delyi) hub = delyi/afi
1767  10 CONTINUE
1768 C
1769 C Set initial guess for h as geometric mean of upper and lower bounds. -
1770  iter = 0
1771  hg = sqrt(hlb*hub)
1772 C If the bounds have crossed, exit with the mean value. ----------------
1773  IF (hub .LT. hlb) THEN
1774  h0 = hg
1775  GO TO 90
1776  ENDIF
1777 C
1778 C Looping point for iteration. -----------------------------------------
1779  50 CONTINUE
1780 C Estimate the second derivative as a difference quotient in f. --------
1781  h = sign(hg, tout - t0)
1782  t1 = t0 + h
1783  DO 60 i = 1, n
1784  60 y(i) = y0(i) + h*ydot(i)
1785  CALL f (n, t1, y, temp, rpar, ipar)

1798 C Stop if the new and previous h values differ by a factor of .lt. 2.
1799 C Stop if four iterations have been done. Also, stop with previous h
1800 C if HNEW/HG .gt. 2 after first iteration, as this probably means that
1801 C the second derivative value is bad because of cancellation error.
1802 C-----------------------------------------------------------------------

1804  hrat = hnew/hg
1805  IF ( (hrat .GT. half) .AND. (hrat .LT. two) ) GO TO 80
1806  IF ( (iter .GE. 2) .AND. (hnew .GT. two*hg) ) THEN
1807  hnew = hg
1808  GO TO 80
1809  ENDIF

1814  80 h0 = hnew*half

1816  IF (h0 .GT. hub) h0 = hub
1817  90 h0 = sign(h0, tout - t0)
1818  niter = iter
1819  ier = 0
1820  RETURN
1821 C Error return for TOUT - T0 too small. --------------------------------
1822  100 ier = -1
1823  RETURN

1825  END
